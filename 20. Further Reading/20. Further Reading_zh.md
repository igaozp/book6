# 延伸阅读

互联网上有大量关于 IPv6 的信息。读者应该注意，并非所有信息都是可靠的。很多时候，这些信息已经过时了，因为 IPv6 最初是在 1990 年代设计的，而整个互联网自那时以来已经发生了很大的变化，IPv6 也随之进行了更新。

IPv6 标准、最佳当前实践和其他技术信息的权威来源是 IETF 的*最新* RFC（征求意见稿）。RFC 可从 [RFC 编辑器](https://www.rfc-editor.org/)免费获取。

*警告：*过时的 RFC 永远不会被修改或删除。在信任 RFC 之前，查看其当前状态至关重要。例如，主要 IPv6 标准的 2017 年版本的当前状态显示在[此信息页面](https://www.rfc-editor.org/info/rfc8200)。每个 RFC 都有一个格式为 `https://www.rfc-editor.org/info/rfcNNNN` 的信息页面，其中 `NNNN` 是 RFC 编号（不带前导零）。如果 RFC 被标记为"Obsoleted by"（已被废弃），通常应该忽略它 - 而是查看替代它的较新 RFC。因此，对 RFC 2460 的任何引用都应被视为对 RFC 8200 的引用。

（偶尔，被废弃的 RFC 可能包含有用的背景信息，或有助于理解旧的协议实现。）

即使没有被废弃，RFC 也可能被一个或多个较新的 RFC "Updated by"（更新）。你需要同时查看这些 RFC。

如果 RFC 被标记为"Proposed Standard"（建议标准）、"Draft Standard"（标准草案）、"Internet Standard"（互联网标准）或"Best Current Practice (BCP)"（最佳当前实践），则它是 IETF 中粗略共识的结果，是一个确定性的规范。但是，这不会覆盖"Obsoleted by"或"Updated by"。

如果它被标记为"Informational"（信息性）、"Experimental"（实验性）或"Historic"（历史性），这些词的含义正如其字面意思。其中一些 RFC 甚至不来自 IETF；它们可能来自 IAB（互联网架构委员会）、IRTF（互联网研究任务组）或其他地方。

任何 RFC 都可能被标记为有*勘误*（拉丁语，表示错误）。请检查它们！通常它们是微不足道的，但有时它们很重要。

在阅读 RFC 时，你可能会注意到错误，或有评论或建议。可以通过信息页面上的 *Submit Errata* 链接报告重大错误。如果你有技术评论或建议，请考虑通过[参与 IETF](https://www.ietf.org/participate/) 或 [IRTF](https://www.irtf.org/) 加入相关工作组。

一个重要的 RFC 是[IPv6 节点要求](https://www.rfc-editor.org/info/bcp220)的最新版本，它引用了许多其他 RFC。然而，在撰写本文时，自节点要求的上次更新以来，IETF 已经发布了至少 12 个更新的 IPv6 RFC。主要 IETF 工作组与 IPv6 相关的文档列在 [6MAN](https://datatracker.ietf.org/wg/6man/documents/) 和 [V6OPS](https://datatracker.ietf.org/wg/v6ops/documents/) 网页上。请注意，这些页面列出了未批准的草案和过时的 RFC 以及当前的 RFC。

在本书的一些情况下，我们引用了未批准的草案（通常称为 Internet-Drafts 或 I-D）。正式而言，使用 I-D 作为参考材料是不恰当的。虽然有时非常有用且最新，但此类草案与 RFC 的地位不同，不应被视为稳定的文档 \[[draft-wkumari-not-a-draft](https://datatracker.ietf.org/doc/draft-wkumari-not-a-draft/)\]。它们尚未经过彻底审查，可能有误，而且很有可能永远不会作为 RFC 发布。文件名以"draft-ietf-"开头的草案已被 IETF 工作组采纳，因此已通过初步审查，但它仍然是草案，可能仍然有误，并且可能永远不会成为 RFC。

名称_不_以"draft-ietf-"开头的草案根据约定的[命名规则](https://authors.ietf.org/naming-your-internet-draft)命名，但它们几乎肯定没有被 IETF 工作组采纳，应谨慎阅读。有关 I-D 的权威信息来源是 [IETF 数据跟踪器](https://datatracker.ietf.org/)。

所有 I-D 都可以接受评论，并包含联系信息。请随时给它们的作者或相关邮件列表发送电子邮件。此图提供了概览：
